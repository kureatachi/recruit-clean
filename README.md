## デプロイメントに関する注意事項

### 1. ファイルプロトコル対応
- すべてのページは `file://` プロトコルで動作するように設定されています
- CDNリンクは `https://` を明示的に指定しています
- 相対パスでリンクが構成されているため、ディレクトリ構造を維持してください

### 2. 必要な外部リソース
- **jQuery**: CDN経由で読み込み（各HTMLファイルの `<head>` 内）
- **Font Awesome**: CDN経由で読み込み（アイコン表示用）
- **Google Fonts**: CDN経由で読み込み（Roboto、Noto Sans JP）

### 3. サーバー環境でのデプロイ
- 静的HTMLサイトとして動作します
- 特にサーバーサイドの処理は不要です
- `.htaccess` やリダイレクト設定は含まれていません

### 4. デプロイ時の注意点
- ディレクトリ構造を変更しないでください
- `css/`、`js/`、`img/` フォルダはルートに配置してください
- すべてのHTMLファイルは同じ階層に配置してください
- 画像ファイルのパスが正しく参照されているか確認してください

### 5. ブラウザ対応
- モダンブラウザ対応（Chrome、Firefox、Safari、Edge）
- レスポンシブデザイン対応（PC、タブレット、スマートフォン）
- スマートフォン向けブレークポイント: 750px以下
- タブレット向けブレークポイント: 751px〜1100px

## ファイル概要

#### index.html
- **説明**: 採用サイトのトップページ
- **主な機能**: メインビジュアル、サイトナビゲーション、サイト全体の入口

- **説明**: 企業情報ページ

- **特徴**: レスポンシブ対応の約束カード（550px-481px範囲で最適化）

- **主な内容**: 営業職、事務職の詳細説明

## 備考

### 1. パス構造
- すべてのファイルパスは相対パスで記述されています
- HTMLファイルから `css/`、`js/`、`img/` への参照は同一階層からの相対パスです
- 画像ファイルは `img/` フォルダ内に整理されています

### 2. レスポンシブデザイン
- ブレークポイント: 1100px、750px
- 1100px以下でモバイルナビゲーション（ハンバーガーメニュー）に切り替わります
- 750px以下でより詳細なモバイル最適化が適用されます

### 3. メニューとナビゲーション
- デスクトップ: 横並びのメニューバー
- モバイル（1100px以下）: ハンバーガーメニュー
- Entryボタンはすべての画面サイズで表示されます

### 4. 特殊な実装
- `#fade` 要素によるページ読み込み制御
- jQuery InViewプラグインによるスクロールアニメーション
- レスポンシブ対応のタイトル（長いタイトルは自動改行）

### 5. ブラウザでの確認
- ローカルファイルとして開く場合: `file://` プロトコルで動作確認可能
- サーバーにアップロード後: HTTP/HTTPSで動作確認

### 6. 更新履歴
- 2024年: 採用サイト専用ディレクトリとして整理
- レスポンシブ対応の改善（550px-481px範囲、750px以下）
- メニュー位置調整、Entryボタンのギャップ修正
- モバイル向けタイトルの横スクロール防止対応

### 7. 注意事項
- jQueryはCDN経由で読み込まれているため、インターネット接続が必要です
- `jquery.modal_scroll.js` はスタブファイルです（実際の機能は未実装）
- 画像ファイルのサイズに注意してください（ページ読み込み速度への影響）

### 8. サポート
- 質問や問題がある場合は、開発チームまでお問い合わせください
- デプロイ前には必ずすべてのページで動作確認を行ってください
